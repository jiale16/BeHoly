# BeHoly 优化审查：从「拦截试探」到「走入光明」

> 审查范围：无障碍检测、分级处置、悔改流程、连胜统计、每日金句、主界面。
> 视角：对照使命「帮助从上瘾中走出来，进入光明中，每一天有圣洁的内在和外在」。

## 一、现状速览（已经做对的）

- **离线铁律守得干净**：Manifest 不声明任何网络权限，检测全部走无障碍文本节点，物理上拿不到像素，无 OCR/模型依赖。架构健康。
- **分级处置设计成熟**：Tier1 踢回桌面+悔改页；Tier2 封禁+锁屏+5 分钟冷静期循环重锁；Tier3 重启。非 Device Owner 自动降级，不崩溃。
- **悔改链路完整**：打断 → 「我愿意悔改归向神」→ 结构化反思表单（心情/方式/反思/避免方案）→ 落盘 JSONL，且支持 SAF 导出/导入跨卸载保留。
- **无障碍关闭劝诫守卫**：用户去设置页想关服务时，在「动手前」给一个转向神的停顿。心思很细。
- **健壮性强**：顶层 try/catch 防进程崩溃、节点 recycle 防泄漏、令牌节流防事件风暴。

结论：这是一个**拦截/中断能力很强**的 App，但在「**建立每日圣洁生活**」这一半使命上几乎是空白。下面 5 个缺口都围绕这一点。

---

## 二、核心缺口（与使命最相关）

### 缺口 1：连胜的「展示命名」可更贴切（逻辑无需改）

> **产品 owner 决策（2026-07-23）**：无障碍一直开启、从未被关掉，本身就是「在试探面前没有缴械」的持续得胜。
> 因此 `StreakStore` 的现有计算（无障碍开启天数，关闭清零）**逻辑正确、无需改**；只需把展示文案从「连续守护 N 天」改为「**得胜 N 天**」之类的正向命名，更准确也更有力量。

**建议（P0，已采纳）**：`MainActivity.updateStreakDisplay()` 展示改为「**已连续得胜 N 天**」（0 天时「今天迈出得胜第一步」）。已写入代码。

**可选补充（非必须，owner 未要求）**：若日后想要「对跌倒也诚实」的维度，可额外加一个独立指标「距上次回转 N 天」（数据已在 `RepentanceStore.createdAt`）。但与「得胜天数」语义不同、**不合并、不替代**。当前不实现，除非你后续想要。

### 缺口 2：没有「每日灵修 / 立志」的主动习惯养成

`DailyVerse.today()` 只出现在常驻通知栏。没有让人每天主动「与神相遇」的入口。
使命说「**每一天**有圣洁的内在和外在」——当前 App 只负责「拦」，不负责「养」。

**建议（P1）**：主界面加「每日灵修」入口 → 今日金句 + 一句引导经文默想 + 「今天我立志行在光明中」一键打卡（记日期，可形成「连续与神相遇天数」这一正向连胜）。把阻断性的工程，补上建造性的日常节律。

### 缺口 3：悔改后的「恩典闭环」缺失

`RepentanceFormActivity.onSave()` 写完记录就 `finish()` 回原 App。没有「你已蒙赦免、平平安安去吧」的安慰时刻，也没有 visibly 重启「光明之旅」计数。

**建议（P0）**：保存后加一个轻量「恩典页」： assurance 经文（如约壹 1:9「我们若认自己的罪，神是信实的，必要赦免」）+ 「今天是你光明中的第 1 天」+ 关闭按钮。闭合属灵循环，让回转有归处的实感。

### 缺口 4：常驻通知只显示金句，浪费最高频触点

`MonitoringService.buildNotification()` 仅放金句。这是用户每天最可能多次看到的界面，却没用来强化「走在光明中第 N 天」。

**建议（P2）**：通知内容轮播 = 金句 + 一行鼓励 + 光明天数，让不打开 App 也能被温柔提醒。

### 缺口 5：检测盲区——纯图文 / 短视频类成瘾内容

无障碍文本检测对纯图片、短视频 feed 失效（如某些短视频/图集类 App 标题不含敏感词）。

**建议（P3，需你拍板）**：维护一份**本地可配置的高风险 App 包名清单**（离线、用户可改），打开即触发 **Tier1 悔改提醒（不封禁）**，作为文本检测的补充。属于策略决策，过度拦截会惹人烦，建议默认保守、留开关。

## 三、工程质量小修

1. **死代码**：`MainActivity` 把 `logs_visible` 写进 `beholy_ui` 偏好，但 `onCreate` 硬写 `logsVisible = false` 且从不读取——写入永不生效。二选一：要么恢复读取，要么删掉写入。
2. **表单文字色硬编码**：`RepentanceFormActivity` 的 CheckBox 用 `android.R.color.white`，依赖暗色表单背景；一旦主题变浅会看不清。建议改用主题属性或布局里定义。
3. **金句池偏小**：`DailyVerse` 仅 31 条、每 31 天循环。可扩充，并为宜每句附一句极短释经/默想引导（配合缺口 2）。
4. **缺「旅程总览」**：主界面只有守护连胜 + 拦截日志。可加一张温和卡片：光明天数 / 蒙恩回转次数 / 灵修打卡天数，给人整体性的盼望，而非只有「又拦了一次」的日志。

## 四、落地优先级

| 优先级 | 缺口 | 直接服务的使命 |
|--------|------|----------------|
| **P0** | 缺口 1 文案改名「得胜天数」（一行）+ 缺口 3 恩典闭环 | 进入光明、圣洁内在（最核心） |
| **P1** | 缺口 2 每日灵修打卡 | 每一天有圣洁的内在 |
| **P2** | 缺口 4 通知增强 + 旅程总览 | 持续被提醒、看见盼望 |
| **P3** | 缺口 5 包名软分级（需决策）+ 小修 | 检测覆盖 / 代码整洁 |
